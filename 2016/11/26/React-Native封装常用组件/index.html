React Native封装常用组件

发表于 2016-11-26   |   分类于 React Native   |     |   阅读次数

最近APP需求改动相当频繁，不少界面基本是要推翻重写。除了产品的“无理”要求之外，代码的可扩展性差、耦合性强也是罪魁祸首之一。这几天重构了大部分界面代码，也封装了一些组件出来，本文就针对其中一个来分享下自己封装的简单思路。相信实践过React Native的朋友，或多或少都自己封装过组件了，这里提到的东西也基本是针对新人，也算是自己的一个总结记录吧。

准备

组件的布局

首先要做这样一个组件，那么可以先考虑第一个问题：如何让组件显示的时候，是覆盖整个屏幕，并且是在最顶层？

我刚接触React Native的时候，Flex布局里面还没有zIndex的概念，后来是增加了，不过我还没有去细看这个属性，所以这里暂时不考虑这个相关的实现方式。查看文档可以知道官方提供了Modal，这个组件就是可以直接覆盖根视图的，并且提供了animationType属性可供设置，如此看来这将会是最好的选择。其实之前的浮层组件，我基本也是用Modal来实现，有兴趣的朋友可以用Modal来封装今天的组件。

本文就不采用这种方式了，改用最基本的View来做。View的布局提供了absolute和releative两种方式，这里采用前者，并且将top、left、bottom和right四个值都设为0，再把该组件放置父组件最后，那么一显示将会覆盖掉父组件（一般是整个屏幕）。

内部状态及与外界交互

组件有显示或隐藏两个状态，也涉及到动画效果，当然还有编辑内容，这些都是要在组件内部自己处理。唯一与外界有关联的，那么就是确定编辑内容后的事件。至于取消事件，只是提供给内部，用来控制是否显示就可以了，与外界无关。本组件触摸背景和取消按钮事件，都将能隐藏组件。这样一来，我们就基本可以定出组件内部需要用的几个state属性：

添加动画

我们一开始就将组件添加到了父组件中，所以在还没显示的时候，其实子组件已经执行了componentDidMount方法，正常且好理解的是，应该将动画和传递进来的文本，在show方法中设置。动画涉及一个透明度和编辑框的缩放，这里透明度用timing来控制opacity，缩放则用spring来控制scale，具体的动画基础知识，可以参考官方文档，这里不做展开。需要注意的是，透明度动画和缩放动画是同时执行，所以需要parallel来组合动画。
