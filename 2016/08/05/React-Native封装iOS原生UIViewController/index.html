React Native封装iOS原生UIViewController

发表于 2016-08-05   |   分类于 React Native   |     |   阅读次数

本Demo是基于公司项目业务需求的。由于项目中要接入IM模块，在自己的一番摸索后，决定采用原生来开发IM模块，这样可以大幅度的利用环信提供的EaseUI，节省开发时间。

背景

项目框架与大部分APP类似，底部由4个Tab组成，其中一个Tab是IM模块，界面顶部是类QQ好友列表顶部的Segment控件，用于控制显示消息列表和好友列表。项目不提供好友管理功能，公司将自己维护一套好友体系，所以好友列表由后台同事直接返回。好友列表将保持使用React Native来实现，而消息列表则计划用原生界面开发。本Demo只显示一个消息列表，以做演示。大家可以在这里查看Demo源码。

实现

1、初始化项目

关于React Native封装iOS原生UI，这里不做细述，可参考官方文档。其实本Demo对iOS原生UIViewController的封装，与原生UI封装方式相差不多。基本思路是在封装的原生UI中，添加需要显示的UIViewController，由原生UI持有该UIViewController。Demo使用到了react-native-scrollable-tab-view和react-native-vector-icons这两个组件，故在新建React Native项目后，安装组件：

3、新建RCTViewManager子类

React Native并不支持对UIViewController的直接封装，但是可以对原生UI进行封装。所以我的思路是先封装一个原生的MessageView，然后在该视图里面，添加UIViewController的view，达到封装UIViewController的目的。在React Native对iOS原生UI的封装中，需要做到：

  • 创建一个RCTViewManager的子类
  • 添加RCT_EXPORT_MODULE()标记宏
  • 实现-(UIView *)view方法

5、添加UIViewController

最后

在React Native中接入原生IM是自己的第一次尝试。一开始项目中的会话列表打算采用React Native实现，然后以模态的方式进入原生的对话窗口（为达到UINavigationController的push效果，可自定义转场动画实现）。后来在react-native官方的Issues里面看到关于这个的讨论Is it possible to push an existing native UIViewController to React-Native’s navigator stack ，就试着在UIView里面添加UIViewController，最后也算摸索出来了，值得记录下，希望也能帮助到其他朋友，实现类似需求。
