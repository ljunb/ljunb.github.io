初次尝试

ListView提供了以下两个方法，可以用来渲染组表头和行内容：

  • renderSectionHeader : (sectionData, sectionID) => renderable

    如果提供了此函数，会为每个小节(section)渲染一个粘性的标题。粘性是指当它刚出现时，会处在对应小节的内容顶部；继续下滑当它到达屏幕顶端的时候，它会停留在屏幕顶端，一直到对应的位置被下一个小节的标题占据为止。

  • renderRow：(rowData, sectionID, rowID, highlightRow) => renderable

    从数据源(Data source)中接受一条数据，以及它和它所在section的ID。返回一个可渲染的组件来为这行数据进行渲染。默认情况下参数中的数据就是放进数据源中的数据本身，不过也可以提供一些转换器。

假如想实现一个展开和折叠的效果，那么最容易想到的一个，就是分组名称即是sectionHeader，提供手势响应方法，改变其组内的行数，达到目的。有了这个思路，可以开始第一次尝试。不过在此之前，还需对数据源进行数据结构的组织，以达到符合ListViewdataSource的要求。ListView默认的提取函数是支持以下形式的数据的：

  1. { sectionID_1: { rowID_1: rowData1, … }, … }
  2. { sectionID_1: [ rowData1, rowData2, … ], … }
  3. [ [ rowData1, rowData2, … ], … ]

第二次尝试

LisgView的构造函数可以接收的参数，不止以上提到的两个，其可接收以下这些参数：

  • getRowData(dataBlob, sectionID, rowID);
  • getSectionHeaderData(dataBlob, sectionID);
  • rowHasChanged(prevRowData, nextRowData);
  • sectionHeaderHasChanged(prevSectionData, nextSectionData);

在返回组头和行内容的方法中，可以自定义数据提取的方式。针对这种列表展开和折叠的需求，该如何来组织数据并返回我们要的数据呢？

展开与折叠都是以组为单位，其相应的标识位应该包含于对应的组的数据源之中。每组返回的是一个数组，换个角度，其实可以在每个数组中，push进去一个拥有标识位的对象，如{hide: false}或是{isShow: true}之类，然后在ListView构造参数的提取数据参数中，自定义返回的数据格式，再在渲染方法中进行判断。
